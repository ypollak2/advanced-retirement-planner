🕵️ Warning Analysis

Root Cause

The warning occurs when the WizardStepSalary component triggers a state update in the ConsoleLogExporter component during its render cycle. This typically happens when:

  • A console.log() call is made during render
  • The ConsoleLogExporter intercepts the log and tries to update its state
  • React detects this cross-component state update during render

🔍 Common Triggers in Income Calculations

Based on the code analysis, potential triggers include:

  • calculateTotalIncome() function calls during render
  • Tax calculation debug logs in AdditionalIncomeTax.getMonthlyAfterTaxAdditionalIncome()
  • Currency validation logs in financial calculations
  • Partner field mapping debug logs

🏥 Recommended Fixes

  1. Move calculations to useEffect: Move income calculations that trigger console logs into useEffect hooks
  2. Debounce console logs: Use setTimeout to defer console logging until after render
  3. Conditional logging: Only log in development and outside render cycles
  4. Memoize calculations: Use useMemo for expensive calculations to prevent re-renders
